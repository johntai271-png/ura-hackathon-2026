# URA Hackathon 2026 – OCR & Product Recognition Pipeline

> **Competition:** [The 2nd URA Hackathon 2026](https://www.kaggle.com/competitions/the-2nd-ura-hackathon)  
> **Task:** Given product images (social-media screenshots, shelf photos, ads), extract OCR text and identify the product/brand name.  
> **Metric:** Composite score = `0.6 × Token-F1(product_name) + 0.4 × (1 − CER(ocr_text))`

---

## 📓 Notebook

**[`ura_hackathon_v4.ipynb`](ura_hackathon_v4.ipynb)** – Final submission notebook (Pipeline v4.0)

### Pipeline overview

```
Image
  └─► PaddleOCR (CPU, latin_PP-OCRv3)   ← OCR engine
          └─► clean_social_ocr()         ← strip @mentions, #hashtags, URLs, timestamps
              └─► BRAND_RULES (46 rules) ← regex-based brand matching
                    └─► _assign_readable_brand_tokens()   ← anti-hallucination filter
                          └─► ProductPredictor (TF-IDF + LogReg, 249 classes)
                                └─► post_process_prediction()   ← Pate/Nestlé/Vinamilk upgrade
                                      └─► submission.csv
```

---

## 🔑 Key Design Decisions

### 1. OCR Engine – PaddleOCR (CPU)
- Uses **`latin_PP-OCRv3`** recognition model + **`ch_PP-OCRv4`** detector.
- Chosen over EasyOCR because the competition's ground-truth OCR is generated by a similar engine → lower Character Error Rate (CER).
- Images pre-processed: resize to ≤ 1 280 px, contrast ×1.35, sharpen.
- Confidence threshold: **0.35** (box is kept only if PaddleOCR confidence > 0.35).

### 2. Social-Noise Filter
- Strips URLs, @mentions, #hashtags, timestamps, and long digit sequences.
- `_is_ocr_noise_only()` – discards OCR results that contain only social-media keywords (TikTok, Reels, Capcut, viral …) with no meaningful product token.

### 3. Brand Rules (46 regex patterns)
- Hierarchical: more-specific patterns are checked before generic ones.
- Handles Vietnamese diacritics, OCR typos (`canfuco` → `canfoco`, `vinamill` → `vinamilk`), and abbreviations.
- Each rule maps to a **canonical brand name** + optional **product-line keywords**.

| Rule group | Examples |
|---|---|
| Ha Long Canfoco + Pate Cột Đèn | `canfoco`, `canfuco`, `ha long canf` |
| Đồ Hộp Hạ Long | `đồ hộp hạ long`, `hophalong` |
| Pate Cột Đèn Hải Phòng | `cotcen`, `cotden`, `cot den hai phong` |
| Dairy & Infant formula | Vinamilk, TH True Milk, Dutch Lady, Nutifood, Abbott, Nestlé NAN, Aptamil, HiPP, Friso, Meiji |
| Coffee & Beverage | Highlands Coffee, The Coffee House |
| Meat & Pate | Vissan, Hafi, CP, Ba Huân, San Hà |

### 4. Anti-Hallucination Token Filter
`_assign_readable_brand_tokens()` — before returning any prediction, every token in the candidate product name must be **found in the OCR text** (exact or compact match). Tokens not grounded in the OCR are silently dropped, preventing the model from inventing brands.

### 5. ML Fallback – `ProductPredictor`
When no rule matches:
1. **Gate classifier** (TF-IDF char n-grams 2–4, LogReg `balanced`): decides if the image contains a product.
2. **Product classifier** (same architecture, 249 classes): predicts the specific product from `train_labels.csv`.
- Trained on 4 892 rows; min 3 samples per class.
- Probability threshold: **0.60** (below that → predict empty string).

### 6. Additional ML Layers (L4 / L5)
- `SklearnPredictor` (L4): higher-capacity LogReg (C=1.5, max_feat=6 000, sublinear TF) trained on normalised product names.
- `KNNPredictor` (L5): cosine-similarity TF-IDF KNN (k=5, threshold=0.60).

### 7. Post-Processing
Fine-grained upgrades applied after any prediction:
- `Pate` → `Pate Cột Đèn Hải Phòng` / `Pate Cột Đèn` / `Đồ Hộp Hạ Long` depending on OCR context.
- `Nestlé` → `Nestlé NAN` / `Nestlé Milo` when corresponding keyword is found.
- `Vinamilk` → `Vinamilk Dielac` when `dielac` / `dulac` is found.

## 🚀 How to run on Kaggle

1. Add the competition dataset to your Kaggle notebook.
2. Set `RUNNING_ON_KAGGLE = True` (already the default).
3. **Important:** Delete any old `checkpoint.csv` before re-running:
   ```python
   import os; os.remove('/kaggle/working/checkpoint.csv')
   ```
4. Run all cells top-to-bottom. The main loop saves progress every 50 images.
5. Download `submission.csv` from `/kaggle/working/` and submit.

**Runtime:** ~21 minutes on a Kaggle CPU kernel (4 threads, 2 006 images, 0.62 s/img).
